## What I do

- Analyze input ideas or product models for comprehensive evaluation
- Identify and document strengths and weaknesses (pros and cons)
- Research and compare cost-effective alternatives and options
- Present structured overviews with clear summaries
- Generate decision frameworks for informed choices

## When to use me

Use this when:
- Evaluating a new product idea or concept
- Comparing multiple product models or solutions
- Assessing cost-benefit of different options
- Making informed decisions between alternatives
- Analyzing competitive products or features
- Reviewing feature trade-offs for a product

## Prerequisites

- Clear description or specification of the idea/product to analyze
- Access to web search for research (use websearch and codesearch tools)
- Understanding of the domain or industry context
- Defined evaluation criteria (optional but helpful)

## Conventions

### Input Format

Provide the following when requesting analysis:
- Product/idea name and brief description
- Target use case or problem being solved
- Any specific constraints (budget, features, timeline)
- Comparison criteria if specific aspects matter

### Analysis Framework

1. **Core Value Proposition** - What problem does it solve?
2. **Feature Comparison** - What capabilities does it offer?
3. **Cost Analysis** - What is the price/value ratio?
4. **Market Position** - How does it compare to alternatives?
5. **Risk Assessment** - What are potential drawbacks?

**Input:**
- Product: Project Management Tool
- Use case: Team collaboration for remote work

- Executive Summary (2-3 sentences)

## Common Issues

### Incomplete Input

**Problem:** Vague or missing context leads to shallow analysis.
**Solution:** Always ask for clarification on:
- Target use case or problem
- Specific constraints or requirements
- Comparison criteria or priorities

### Overlooking Alternatives

**Problem:** Focusing only on well-known options.
**Solution:** Research both popular and niche alternatives:
- Use web search for comprehensive market view
- Include open-source options
- Consider emerging solutions

### Bias in Analysis

**Problem:** Favoring certain products without objective criteria.
**Solution:** Apply consistent evaluation criteria:
- Use same metrics across all options
- Consider both quantitative and qualitative factors
- Acknowledge limitations and uncertainties

### Cost Oversimplification

**Problem:** Comparing only sticker prices without total cost.
**Solution:** Consider:
- Implementation costs
- Training and onboarding
- Maintenance and support
- Integration expenses
- Hidden or future costs
